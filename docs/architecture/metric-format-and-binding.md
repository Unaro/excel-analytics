# Формат метрики и привязка колонок дашборда

Статус: реализовано (фазы 1–2). Задача: убрать дублирование настройки
формата и упростить привязку групп к дашборду.

## Проблема

Формат, decimals и единица задавались в трёх местах: шаблон, метрика
группы, колонка дашборда. Один шаблон в 5 группах → формат настраивался
5 раз, потом ещё на дашборде. Привязка колонок к группам была ручной
(таблица выпадашек: каждая ячейка группа×колонка).

## Решение: шаблон — единственный источник

### Фаза 1 — формат на шаблоне

- `MetricTemplate`: `prefix`/`suffix` сведены к одному `unit`; формат,
  decimals и unit задаются в TemplateForm (раньше захардкожены).
- Наследование: `buildVirtualMetric`, `kpi-compiler`, KPI-грид берут
  формат/decimals/unit из шаблона. Единица метрики группы (`GroupMetric.unit`)
  — опциональный override (placeholder показывает шаблонную).
- Удалены мёртвые `customDisplayFormat`/`customDecimalPlaces`.
- Миграции: template-store v1→v2 (suffix/prefix→unit),
  indicator-group v2→v3 (чистка custom-полей).

### Фаза 2 — колонка дашборда = шаблон

- Новый тип `DashboardColumn` (хранимая колонка): `templateId` + `colorConfig`
  (пороги — дашборд-специфичны) + `order`. Формат/имя/единица НЕ хранятся —
  выводятся из шаблона. `VirtualMetric` остался транзитным носителем для
  движка (поля формата обязательны).
- Чистый слой `shared/lib/utils/dashboard-columns.ts` (+ тесты):
  - `buildEffectiveColumn(column, template)` → VirtualMetric с форматом
    из шаблона (для таблицы и движка);
  - `resolveColumnMetricId(group, templateId, override?)` → авто-привязка
    к метрике группы по шаблону, override приоритетнее;
  - `resolveDashboardGroupsConfig(...)` → материализует привязки
    `virtualMetricId→metricId` для движка (контракт движка НЕ изменён);
  - `resolveColumnTemplateId(...)` → ленивая миграция старых колонок
    (выводит templateId из существующих привязок).
- `use-dashboard-computation` строит эффективные колонки + резолвит
  привязки на лету; движок и таблица получают формат из шаблона.
- Билдер: «добавить колонку» = выбрать шаблон (из тех, что используют
  добавленные группы); добавил группу → ячейки привязались сами;
  ручной выбор в ячейке появляется только при дубле шаблона в группе.

## Миграция старых дашбордов (ленивая)

Старые колонки хранят формат и привязки по `metricId`. При просмотре:
templateId выводится из привязок (мажоритарно), формат берётся из шаблона,
старые привязки работают как override — поведение сохраняется без
обязательного пересохранения. При следующем сохранении в билдере
templateId записывается в колонку, завершая миграцию. Версия persist
дашборда не менялась — `DashboardColumn` терпит legacy-поля при чтении.

## Граница, которую держали

Движок вычислений (`worker`, `pg-engine`) и формат хранения ключей
(`${groupId}__${metricId}`) НЕ трогались — менялся только слой сборки
привязок и формата перед движком и в UI.
