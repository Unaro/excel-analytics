# Roadmap / бэклог идей

Зафиксировано 2026-06-12 по итогам рефакторинга и аудитов.
Приоритеты: 🔥 рекомендовано на ближайшую итерацию · ⭐ высокая ценность · ◌ фоном.

## Быстрые победы (код наполовину готов)

- 🔥 **Экспорт в Excel** — breakdown-таблицы, pivot «иерархия × время»,
  таблица дашборда. Библиотека `xlsx` уже в зависимостях (импорт),
  нужно обратное направление.
- 🔥 **Период-к-периоду** — поля `previousValue` / `change` / `changePercent`
  в `ComputedMetricValue` (shared/lib/types/computation.ts) заведены,
  но никем не заполняются. С группировкой по датам сравнение
  «текущий месяц vs прошлый» на KPI-карточках — естественный шаг.
- ⭐ **Фильтр по диапазону дат** — `ComputationContext.dateRange` существует
  и не используется. Date-picker над дашбордом + WHERE в query-compiler.
- ⭐ **Σ в pivot для AVG/MAX/MIN** — протащить тип агрегации в метаданные
  виртуальной метрики; для AVG показывать среднее, для MAX/MIN экстремум,
  а не сумму (текущая Σ в TimeBreakdownSection осмысленна только для
  SUM/COUNT).

## Хвосты аудитов (см. AUDIT-CORE-MODULES.md, AUDIT-UI-COMPONENTS.md)

- 🔥 **№11 — KPI в основной запрос**: смена фильтра гоняет два SQL по одним
  данным (дашборд + KPI-группа). Для PG — двойной сетевой трафик.
  kpi-compiler уже создаёт совместимые структуры.
- ⭐ **№13 — hierarchyFilters в persist**: «застрявшие» фильтры после
  перезагрузки. Лечится `partialize` (ключи localStorage не переименовывать!).
- ◌ №12 — кэш DESCRIBE в воркере (инвалидация в IMPORT_*/DROP_TABLE).
- ◌ №14 — deps-разнобой: `widgets.length` (use-kpi-calculation),
  `currentPath` (use-group-breakdown) — только примитивы/хеши.
- ◌ №15/16 — линейные find в компиляторе, откат батча при ошибке импорта,
  неиспользуемый immer, dispose() в cleanup.
- ◌ Дубли: `toLocaleString('ru-RU')` в ~10 файлах → общий formatNumber;
  `buildVirtualMetrics` повторяется в worker и pg-engine → общий хелпер.
- ◌ Адаптивность: поповеры `w-[350px]`, `GroupAdder` w-50,
  `max-w-[200px]` у инпута названия метрики.

## Среднесрочные фичи

- ⭐ **Сохранение состояния просмотра** — выбранные метрики, размерность дат,
  состав серий чарта сбрасываются при уходе со страницы. Persist
  per dashboard/group.
- ⭐ **Публичные read-only дашборды** — поле `Dashboard.isPublic` есть в типе
  и не работает. Режим «поделиться ссылкой» без редактирования.
- ◌ **Выбор дата-колонки** — сейчас берётся первая с классификацией «Дата»;
  нужен селектор при нескольких.
- ◌ **Алёрты по порогам** — правила окрашивания уже описывают «плохо/хорошо»;
  после обновления PG-данных показывать сводку «N метрик вышли за пороги».

## Инженерное здоровье
